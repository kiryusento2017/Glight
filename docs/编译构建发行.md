# 编译 / 构建 / 发行流程

> 本机 Go 不在 PATH，下文 `go` 代表完整路径 `C:\Open Source Projects\go\bin\go.exe`。
> goversioninfo 在 `C:\Users\<你>\go\bin\goversioninfo.exe`。

## 一句话原则

**调试用 `go run .`（不产生 exe、带控制台看输出）；凡是要一个 exe 文件，只有一条命令，所有防护一次带齐。**

不存在「调试版 exe」——这样就绝不会误发带本机特征或带黑窗的版本。

---

## 场景一：日常调试（改代码、看效果、查崩溃）

```powershell
go run .
```

- **带控制台黑窗**：panic 堆栈、`fmt.Println` 等输出都看得到
- **不产生 exe 文件**：跑完即弃，不污染目录
- 改外观参数不用重编：直接编辑 `glass-tuning.json`，存盘即热重载

跑测试：

```powershell
go test ./...
```

---

## 场景二：编译出 exe（本地试装 或 发行，同一条命令）

```powershell
# dist 不存在先建：mkdir dist
go build -trimpath -buildvcs=false -ldflags="-H windowsgui" -o dist\claude-traffic-light.exe .
```

产物在 `dist\claude-traffic-light.exe`。这条命令把四件防护一次带齐：

| 参数 | 作用 |
|---|---|
| `-trimpath` | 清掉 exe 里的本机绝对路径（盘符 / Windows 用户名 / 项目目录），换成模块相对路径 |
| `-buildvcs=false` | 去掉嵌入的 git 提交号 / 时间 / dirty 标记 |
| `-ldflags="-H windowsgui"` | 无控制台黑窗（GUI 程序该有的样子） |
| `-o dist\...` | 统一输出到 `dist/`（已 gitignore），与源码隔离 |
| （自动） | 链接 `rsrc_windows_amd64.syso` = 图标 + 版本信息（产品名/版本/署名），让裸 exe 有正规身份、降杀软误报 |

**严禁**：加 `-s -w`（strip 符号反而触发 Wacatac 误报）、加壳（UPX）。

构建后自检本机特征已清零：

```powershell
# 应为 0
Select-String -Path dist\claude-traffic-light.exe -Pattern "你的用户名" -Encoding Byte 2>$null | Measure-Object | Select Count
```

## 场景三：正式发行到 GitHub（防报毒完整流程）

1. **编译**：跑场景二那条命令，产物在 `dist\`
2. **自查**：把 `dist\claude-traffic-light.exe` 传 [VirusTotal](https://www.virustotal.com)，记录哪些引擎报、报什么名
3. **提交代码**：`git add` / `git commit` / `git push`
4. **发 Release**：GitHub Releases 上传 exe，附 `SHA256` + 一句「无签名 Go 程序的已知误报，More info → Run anyway」说明 + VirusTotal 链接
5. **误报申诉**（免费，对 Defender 最有效）：
   - Defender → https://www.microsoft.com/en-us/wdsi/filesubmission
   - SmartScreen → https://feedback.smartscreen.microsoft.com
   - ⚠️ 申诉针对**具体文件哈希**，每个新版本 exe 都要重新提交

> 详细防报毒 SOP（含为什么不走签名、信誉累积）见 [发布清单.md](发布清单.md)。

## 什么时候要重新生成 syso？

`rsrc_windows_amd64.syso`（图标 + 版本信息）是**预生成、已入库**的，平时构建不用碰。**只在改图标 / 版本号 / 署名时**重新生成：

```powershell
# 1. 改 versioninfo.json（版本号或署名）
# 2. 重新合成 syso（图标 + 版本信息 → 一个文件）
goversioninfo -icon=claude-traffic-light.ico -o=rsrc_windows_amd64.syso
# 3. 重新编译（场景二命令）
```

> ⚠️ **发行前必须先确认版本号**：构建前务必问清楚当前应发哪个版本（Major.Minor.Patch），
> 不许沿用旧版本号直接构建。版本号改完先改 `versioninfo.json`，再重生成 syso，再编译。

- 工具装一次：`go install github.com/josephspurrier/goversioninfo/cmd/goversioninfo@latest`
- **严禁**再生成第二个含资源段的 `.syso` → 链接报 `too many .rsrc sections`
- 当前署名：`versioninfo.json` 里 `CompanyName` / `LegalCopyright` = **终末诗篇**（中文，会显示在 exe 右键→属性→详细信息）
